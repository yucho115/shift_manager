# shift_manager

JavaのServlet・JSPを用いて作成したシフト管理アプリケーション`shift_web`を、Djangoを用いて改善・再構築したWebアプリケーションです。

`shift_web`で実装したユーザー登録、ログイン、シフト提出、シフト確認などの基本機能を引き継ぎながら、Djangoの認証機能やモデル機能を利用して、セキュリティ、保守性、拡張性を向上させました。

従業員はカレンダーから希望する勤務日時を提出でき、管理者は従業員から提出されたシフトを確認・管理できます。ユーザーの役割に応じて利用できる画面や機能を分けることで、実際の店舗や組織での利用を想定した構成にしています。

## 機能

**1. ユーザー登録・ログイン**<br>

ユーザーはアカウントを作成し、登録した情報を利用してログインできます。

Djangoの認証機能を利用しているため、パスワードはハッシュ化された状態で保存されます。

Java版の`shift_web`では、ServletとDAOを用いて独自にログイン処理を実装していましたが、`shift_manager`ではDjangoの認証機能を利用することで、より安全で管理しやすい構成に改善しています。

**2. 管理者と従業員の役割を管理する**<br>

ユーザーには、シフトを管理する管理者と、希望シフトを提出する従業員の役割があります。

ログインしているユーザーの役割に応じて表示する画面や利用できる機能を分けています。

これにより、従業員が管理者用の機能を利用することを防ぎ、それぞれのユーザーに必要な操作だけを提供します。

**3. 従業員がシフトを提出する**<br>

従業員はカレンダーから勤務を希望する日付を選択し、勤務可能な時間帯を入力してシフトを提出できます。

提出されたシフトには、従業員、勤務日、勤務開始時刻、勤務終了時刻などの情報が保存されます。

**4. 提出済みのシフトを確認する**<br>

従業員は、自分が提出したシフトを一覧またはカレンダー上で確認できます。

ログインしているユーザーに関連付けられたシフトだけを取得するため、他の従業員の希望シフトが表示されないようにしています。

**5. 提出したシフトを編集する**<br>

従業員は、すでに提出したシフトの勤務日や勤務時間を変更できます。

変更した内容はデータベースに保存され、シフトの表示にも反映されます。

**6. 提出したシフトを削除する**<br>

従業員は、勤務できなくなった日などの不要な希望シフトを削除できます。

削除できるシフトをログイン中のユーザーが登録したものに限定することで、他のユーザーのシフトが変更されないようにしています。

**7. 管理者が従業員のシフトを確認する**<br>

管理者は、従業員から提出された希望シフトを確認できます。

日付や従業員ごとにシフトを確認できるため、勤務可能な従業員を把握しやすくしています。

Java版の`shift_web`では指定した日付をもとにシフトを検索する機能を実装していましたが、`shift_manager`ではDjangoのモデルを利用して、シフト情報をより管理しやすい形に改善しています。

**8. シフト情報をデータベースに保存する**<br>

ユーザー情報や提出されたシフト情報はデータベースに保存されます。

アプリケーションを終了した後も、登録したユーザーやシフトの情報が保持されます。

DjangoのORMを利用してデータベースを操作することで、Java版で使用していたSQLやDAOを個別に記述する構成よりも、簡潔で保守しやすい実装にしています。

## 技術スタック

* フロントエンド：HTML、CSS、JavaScript、Bootstrap 5
* バックエンド：Django（Python）
* データベース：PostgreSQL
* 認証：Django Authentication
* UI：django-bootstrap5
* データベース操作：Django ORM


## `shift_web`から改善した点

`shift_manager`は、JavaのServlet・JSPを用いて作成した`shift_web`の基本機能を引き継ぎながら、より現実的なWebアプリケーションとして再構築しています。

### Djangoの認証機能を利用

`shift_web`ではログイン処理やユーザー情報の管理をServlet、セッション、DAOを利用して個別に実装していました。

`shift_manager`ではDjangoの認証機能を利用することで、ログイン状態の管理やパスワードのハッシュ化をフレームワークに任せ、セキュリティを向上させました。

### Django ORMによるデータベース操作

`shift_web`では、JDBC、`PreparedStatement`、DAOを利用してSQLを実行していました。

`shift_manager`ではDjango ORMを利用し、Pythonのコードからユーザーやシフトのデータを操作できるようにしました。

これにより、SQLを直接記述する箇所を減らし、データベース処理の可読性と保守性を向上させています。

### ユーザーごとのアクセス制御

ログインしているユーザーに関連付けられたシフトだけを表示・編集・削除できるようにしています。

単に画面を分けるだけではなく、サーバー側でも操作対象のユーザーを確認することで、他のユーザーのデータに不正にアクセスされることを防ぎます。

### 管理者と従業員の権限を分離

管理者と従業員で利用できる機能を分けています。

従業員は自分のシフトを提出・管理し、管理者は従業員から提出されたシフトを確認するという、実際の業務に近い役割分担を実現しています。

### アプリケーションごとに役割を分割

シフトに関する処理を`shifts`、ユーザーに関する処理を`accounts`として分けています。

Java版よりも機能ごとの責任範囲を明確にし、機能の追加や修正を行いやすい構成にしています。

### 入力フォームと入力値検証を改善

DjangoのFormやModelFormを利用して、入力内容の受け取りと検証を行います。

必須項目の不足や不正な値をサーバー側で検出できるため、フォームから送信された値を個別に確認していたJava版よりも、安全で統一的な入力処理を実現しています。

### 画面の操作性を改善

Bootstrapを利用して画面のレイアウトを統一し、シフトの提出や確認を直感的に行えるようにしています。

カレンダーやフォームを利用することで、日付と時間を選びやすい画面を目指しています。

## 工夫した点

* Djangoの認証機能を利用して、ログイン状態とユーザー情報を安全に管理した
* パスワードをハッシュ化して保存し、データベースに平文のパスワードが残らないようにした
* シフトの作成、一覧表示、編集、削除というCRUD機能を実装した
* 管理者が従業員を招待できる機能を追加した
* 管理者が従業員の希望シフトを承認できる機能を追加した
* Django ORMを利用し、SQLを直接記述せずにデータベースを操作できるようにした
* FormやModelFormを利用し、入力処理と入力値検証を統一した
* シフト機能とユーザー管理機能を別のアプリケーションに分け、役割を明確にした
* Bootstrapを利用して画面のデザインを統一した
* ユーザーがURLを直接入力した場合でも、権限のないデータを操作できないようにした
